# DMR Detection - Usage Guide

## Overview
Differentially methylated region (DMR) detection aggregates per-CpG methylation into contiguous regions that differ between conditions. The decision-grade framing: a DMR is DEFINED by a chain of arbitrary thresholds (min-CpGs, max-gap, delta-beta, q) plus a smoothing bandwidth, and the region p-value is only honest if its null was generated by re-running that selection under permuted labels. dmrseq is the headline caller built to do this; DSS, methylKit tiles, bsseq BSmooth, DMRcate, metilene, and comb-p combine-and-correct but do not re-select, so region q-values are not comparable across tools and cross-tool overlap is the real evidence. A separate single-sample domain-segmentation step (PMDs/UMRs/LMRs via MethylSeekR) must run before focal calling on cancer, aging, placenta, or cultured-cell genomes.

Conceptual prerequisites and notes:
- Input is a per-CpG methylation+coverage table: Bismark `.cov`/cytosine reports for sequencing callers, a beta/M matrix for DMRcate array mode, or long-read modkit bedMethyl counts piped from long-read-sequencing/nanopore-methylation.
- Annotation packages (annotatr, TxDb) are large and genome-build-specific; the build must match the alignment genome.
- dmrseq and the sequencing callers need count data, not betas; arrays use DMRcate or comb-p.
- Run per-CpG quality checks and the methylKit object-model import (methylkit-analysis) before region calling.

## What the Agent Will Do
1. Identify the assay (WGBS/RRBS/array/long-read), replicate count, and coverage to pick a caller from the decision tree.
2. For cancer/aging/placenta/cultured samples, segment single-sample domains (PMDs/UMRs/LMRs) before focal calling and screen for PMD-background DMRs.
3. Build the appropriate input object (BSseq counts, methylKit object, or beta/M matrix) without pre-smoothing where the caller smooths internally.
4. Run the headline caller (dmrseq) with documented thresholds and a selection-aware region FDR.
5. Corroborate with a second caller and report cross-tool overlap rather than a single tool's count.
6. Annotate DMRs to features, flag multiple-genes-per-DMR ambiguity, and hand CpG-bias-aware enrichment to pathway-analysis/go-enrichment.

## Tips
- dmrseq is the headline for WGBS region inference because its permutation null re-runs the region selection; the others give a screening or combined q, not a selection-corrected one.
- Never pre-smooth before dmrseq - it smooths the methylation difference internally.
- DMRcate has two entry points: `cpg.annotate('array', ...)` for arrays and `sequencing.annotate()` with `C=50` for WGBS; the array `C=2` over-smooths sequencing data.
- Region thresholds (delta-beta, min-CpGs, max-gap, q) are conventions; report and justify them and check sensitivity rather than copying one config.
- Cross-tool DMR counts are not comparable; intersect two callers and report the overlap.
- On cancer/aging/placenta/cultured-cell genomes, segment PMDs first; a focal caller manufactures fake hypo-DMRs from the megabase PMD background.
- DMR-to-gene mapping is a modeling choice (nearest TSS vs overlap vs within-X-kb) that changes the gene list; methylation enrichment needs a CpG-bias-aware method (missMethyl gometh/goregion).
